## addendum — cost-management enablement: confirmed console-only + guidance shipped

verified against real AWS after the wisher applied the `provision/aws.auth` perms:

- **the IAM perms worked** — the identity-policy denial is gone; a real `setBudget` reached AWS
  and created a budget, then surfaced a DEEPER, non-IAM wall:
  > `AccessDeniedException: Account … is a linked account. To enable budgets for your account,
  > ask the payer account to enable budgets first.`
- **this is the management-account-gravity wall the vision predicted — and it is NOT declarable.**
  AWS exposes NO api / CloudFormation / SDK to enable Cost Explorer / Budgets; it is a one-time,
  console-only, irreversible activation on the payer account ("Launch Cost Explorer", which also
  enables Budgets). confirmed via AWS docs (`ce-enable.html`: "you can't enable Cost Explorer
  via the API") + the CFN CE provider (only CostCategory / AnomalyMonitor / AnomalySubscription
  exist — none enables the service). this closes the vision's `[research]` item on the
  acceptance / dogfood cred path: the enablement is inherently a human step.
- **so we SHIP the guidance pattern instead** — read the state via api, guide the human by hand:
  - `getCostManagementGuidanceError` (a pure detector) recognizes the "please enable" signal
    (distinct from an iam denial) and returns a `BadRequestError` with the exact console steps.
  - wired as SDK middleware on BOTH `getAwsBudgetsClient` + `getAwsCostExplorerClient`, so one
    chokepoint each covers every read + write. any budget/ce op now emits actionable guidance
    rather than the cryptic aws error. verified: the real `budget.journey` run now throws the
    guided `BadRequestError` with the "Launch Cost Explorer" steps.
  - this is declarative-friendly: `get` reads the state -> if off, guides -> once the human flips
    the switch, the next `get` succeeds and the resource converges to KEEP.
- **still one human step, now self-explained**: enable Cost Explorer once in the payer account
  (the guidance spells out how). after that, budgets/anomaly resources apply for real.
