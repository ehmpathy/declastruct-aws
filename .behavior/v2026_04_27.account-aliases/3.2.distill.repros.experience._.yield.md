# distill: experience reproductions

## experience reproductions

| experience | entry point | user actions | expected outcome | test type |
|------------|-------------|--------------|------------------|-----------|
| set alias for new account | `setAccountAlias` | call with upsert | alias created, console shows alias | integration |
| get current alias | `getAccountAlias` | call with `by: { auth: true }` | returns alias or null | integration |
| update alias (rebrand) | `setAccountAlias` | call with different alias | old deleted, new created | integration |
| remove alias | `delAccountAlias` | call with `by: { auth: true }` | alias removed | integration |
| idempotent set | `setAccountAlias` | call twice with same alias | second call is no-op | integration |
| invalid alias format | `setAccountAlias` | call with invalid chars | BadRequestError before AWS call | unit |
| alias taken in partition | `setAccountAlias` | call with taken alias | clear error with partition explanation | integration |

## journey test sketches

### journey 1: first-time alias setup

**scenario**: platform engineer provisions new account and sets alias

#### step table

| step | action | user sees |
|------|--------|-----------|
| t0 | before any changes | `getAccountAlias` returns `null` |
| t1 | set alias | `setAccountAlias` returns `{ account: { id }, alias: 'ahbode-prep' }` |
| t2 | verify | `getAccountAlias` returns same alias |

#### t0 before state (snapshot target)

```ts
// input
const aliasBefore = await getAccountAlias({ by: { auth: true } }, context);

// output
null
```

#### t1 set alias (snapshot target)

```ts
// input
const aliasAfter = await setAccountAlias(
  { upsert: { alias: 'declastruct-test-alias' } },
  context,
);

// output
{
  account: { id: '123456789012' },
  alias: 'declastruct-test-alias'
}
```

#### t2 verify persistence (snapshot target)

```ts
// input
const aliasRead = await getAccountAlias({ by: { auth: true } }, context);

// output
{
  account: { id: '123456789012' },
  alias: 'declastruct-test-alias'
}
```

## reproduction feasibility

### test utilities available

- `getSampleAwsApiContext()` — real AWS credentials for integration tests
- `getMockedAwsApiContext()` — fake credentials for unit tests
- `given/then` from `test-fns` — BDD structure
- `useBeforeAll` — shared setup

### setup required

- integration tests need IAM permissions: `iam:CreateAccountAlias`, `iam:DeleteAccountAlias`, `iam:ListAccountAliases`
- test alias must be unique per run (use timestamp or UUID suffix)
- cleanup: delete test alias in `afterAll`

### concrete test sketch

```ts
// accountAlias.play.integration.test.ts
describe('accountAlias journey', () => {
  const context = useBeforeAll(() => getSampleAwsApiContext());
  const testAlias = `declastruct-test-${Date.now()}`;

  afterAll(async () => {
    await delAccountAlias({ by: { auth: true } }, context).catch(() => {});
  });

  given('[case1] new account without alias', () => {
    when('[t0] before any changes', () => {
      const aliasBefore = useThen('alias is null', async () =>
        getAccountAlias({ by: { auth: true } }, context),
      );

      then('input/output matches snapshot', () => {
        expect(aliasBefore).toMatchSnapshot();
      });
    });

    when('[t1] alias is set', () => {
      const aliasAfter = useThen('alias is created', async () =>
        setAccountAlias({ upsert: { alias: testAlias } }, context),
      );

      then('alias matches expected', () => {
        expect(aliasAfter.alias).toBe(testAlias);
      });

      then('input/output matches snapshot', () => {
        expect(aliasAfter).toMatchSnapshot();
      });
    });

    when('[t2] alias is read', () => {
      const aliasRead = useThen('alias persists', async () =>
        getAccountAlias({ by: { auth: true } }, context),
      );

      then('alias matches what was set', () => {
        expect(aliasRead?.alias).toBe(testAlias);
      });
    });
  });
});
```

## gaps

### console verification cannot be automated

**what is blocked**: verify alias appears in AWS console header

**what is required**: manual verification or browser automation (e.g., Playwright with AWS console)

**blocker status**: not a blocker — API correctness implies console correctness. manual verification at release.

### partition collision test requires known-taken alias

**what is blocked**: test `EntityAlreadyExistsException` path

**what is required**: either find a commonly-taken alias, or mock the SDK in unit test

**blocker status**: not a blocker — unit test with mocked SDK suffices for error path validation
