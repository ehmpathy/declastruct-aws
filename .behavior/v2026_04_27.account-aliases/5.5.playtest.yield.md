# playtest: account alias

## prerequisites

- AWS credentials configured with IAM permissions:
  - `iam:CreateAccountAlias`
  - `iam:DeleteAccountAlias`
  - `iam:ListAccountAliases`
- keyrack unlocked: `rhx keyrack unlock --owner ehmpath --env test`
- dependencies installed: `npm ci`

## sandbox

- no file operations required for this playtest
- all operations target AWS IAM (account alias is account-level, not file-level)

## happy paths

### 1. complete lifecycle (create → read → update → delete)

**acceptance test coverage:** `accountAlias.acceptance.test.ts` case1

```bash
# run the acceptance test to verify lifecycle
npm run test:acceptance -- --testNamePattern='complete lifecycle'
```

**expected:**
- test passes
- output shows 7 steps: get null → set → get alias → set idempotent → del → get null → del idempotent

**pass:** all assertions pass, snapshots match
**fail:** any assertion fails or snapshot differs
