# research: internal production code patterns

## summary

analysis of production codepath patterns in declastruct-aws relevant to account alias implementation.

## domain object patterns

### [REUSE] DomainEntity base class

domain objects extend `DomainEntity<T>` from `domain-objects` package [1][2].

```ts
// src/domain.objects/DeclaredAwsOrganization.ts:69-100
export class DeclaredAwsOrganization
  extends DomainEntity<DeclaredAwsOrganization>
  implements DeclaredAwsOrganization
{
  public static primary = ['id'] as const;
  public static unique = ['managementAccount'] as const;
  public static metadata = ['id', 'arn'] as const;
  public static readonly = [] as const;
  public static nested = { ... };
}
```

### [REUSE] name convention: `DeclaredAws*`

all domain objects follow `DeclaredAws{ServiceOrConcept}` pattern [1][2][3].

examples from codebase:
- `DeclaredAwsOrganization` [1]
- `DeclaredAwsOrganizationAccount` [2]
- `DeclaredAwsSsoInstance` [3]
- `DeclaredAwsIamRole` (not `DeclaredAwsIamIamRole`) [4]

> "the domain model should reflect what things *are*, not which sdk manages them" — vision document

### [REUSE] primary vs unique key semantics

| key type | semantic | example |
|----------|---------|---------|
| primary | aws-assigned identifier | `['id']` or `['arn']` |
| unique | user-specified identity | `['email']` or `['managementAccount']` |

from DeclaredAwsOrganizationAccount [2]:
```ts
// src/domain.objects/DeclaredAwsOrganizationAccount.ts:119-125
public static primary = ['id'] as const;      // 12-digit account number
public static unique = ['email'] as const;    // globally unique email
```

### [REUSE] metadata and readonly fields

metadata = aws-assigned on creation, readonly = aws-derived current state [2][3].

```ts
// src/domain.objects/DeclaredAwsOrganizationAccount.ts:131-137
public static metadata = ['id', 'arn'] as const;
public static readonly = ['state', 'joinedMethod', 'joinedAt'] as const;
```

### [REUSE] RefByPrimary and RefByUnique for references

domain objects reference others via typed refs [1][2][3].

```ts
// src/domain.objects/DeclaredAwsOrganization.ts:59
managementAccount: RefByPrimary<typeof DeclaredAwsOrganizationAccount>;

// src/domain.objects/DeclaredAwsOrganizationAccount.ts:58
organization: RefByUnique<typeof DeclaredAwsOrganization>;
```

## domain operation patterns

### [REUSE] operation names: `getOne*`, `set*`, `del*`

operations follow strict name convention [5][6][7].

| verb | semantic | examples |
|------|-----------|----------|
| `getOne*` | retrieve single resource | `getOneOrganization`, `getOneSsoInstance` |
| `getAll*` | retrieve collection | `getAllOrganizationAccounts` |
| `set*` | findsert or upsert | `setOrganization`, `setSsoInstance` |
| `del*` | delete (idempotent) | `delOrganization` |

### [REUSE] `(input, context)` signature pattern

all operations use `(input, context)` pattern with typed context [5][6][7].

```ts
// src/domain.operations/organization/getOneOrganization.ts:31-42
export const getOneOrganization = asProcedure(
  async (
    input: { by: PickOne<{ auth: true; primary: ...; unique: ...; ref: ...; }> },
    context: ContextAwsApi & VisualogicContext,
  ): Promise<HasReadonly<typeof DeclaredAwsOrganization> | null> => { ... }
);
```

### [REUSE] `asProcedure` wrapper

operations wrapped with `asProcedure` from `as-procedure` [5][6][7].

```ts
import { asProcedure } from 'as-procedure';
export const getOneOrganization = asProcedure(async (input, context) => { ... });
```

### [REUSE] `PickOne<>` for mutually exclusive options

input uses `PickOne<>` for mutually exclusive options [5][6][7].

```ts
// src/domain.operations/organization/getOneOrganization.ts:34-40
by: PickOne<{
  auth: true;                                           // singleton pattern
  primary: RefByPrimary<typeof DeclaredAwsOrganization>;
  unique: RefByUnique<typeof DeclaredAwsOrganization>;
  ref: Ref<typeof DeclaredAwsOrganization>;
}>
```

### [REUSE] `by: { auth: true }` singleton pattern

for account-scoped resources, use `by: { auth: true }` [5].

```ts
// src/domain.operations/organization/getOneOrganization.ts:67-68
// by auth: return directly (no validation needed)
if (input.by.auth) return found;
```

### [REUSE] findsert vs upsert semantics

`set*` operations distinguish findsert (find-or-create) from upsert (find-or-create-or-update) [6].

```ts
// src/domain.operations/organization/setOrganization.ts:26-31
input: PickOne<{
  findsert: DeclaredAwsOrganization;
  // Note: upsert not supported — organizations cannot be updated
}>
```

### [REUSE] idempotent delete pattern

delete operations return success if already deleted [7].

```ts
// src/domain.operations/organization/delOrganization.ts:56-58
// if not found, return success (idempotent)
if (!foundBefore) return { deleted: true };
```

### [REUSE] ref resolution via type guards

`by.ref` resolved to primary or unique via type guards [5][7].

```ts
// src/domain.operations/organization/getOneOrganization.ts:44-50
if (input.by.ref) {
  if (isRefByUnique({ of: DeclaredAwsOrganization })(input.by.ref))
    return getOneOrganization({ by: { unique: input.by.ref } }, context);
  if (isRefByPrimary({ of: DeclaredAwsOrganization })(input.by.ref))
    return getOneOrganization({ by: { primary: input.by.ref } }, context);
  BadRequestError.throw('ref is neither unique nor primary', { input });
}
```

## cast function patterns

### [REUSE] `castInto*` name pattern

cast functions follow `castInto{DomainObject}` pattern [8].

```ts
// src/domain.operations/organization/castIntoDeclaredAwsOrganization.ts:15-29
export const castIntoDeclaredAwsOrganization = (
  input: Organization,
): HasReadonly<typeof DeclaredAwsOrganization> => {
  return assure(
    DeclaredAwsOrganization.as({ ... }),
    hasReadonly({ of: DeclaredAwsOrganization }),
  );
};
```

### [REUSE] `HasReadonly<>` return type

cast functions return `HasReadonly<typeof DomainObject>` [8].

```ts
import { type HasReadonly, hasReadonly } from 'domain-objects';
// ...
return assure(result, hasReadonly({ of: DeclaredAwsOrganization }));
```

### [REUSE] `assure` for runtime validation

use `assure` from `type-fns` to validate presence [8].

```ts
import { assure, isPresent } from 'type-fns';
// ...
id: assure(input.Id, isPresent),
```

## context patterns

operations depend on `ContextAwsApi` for credentials [9].

```ts
// src/domain.objects/ContextAwsApi.ts:6-20
```

operations also depend on `VisualogicContext` for observability [5][6][7].

```ts
import type { VisualogicContext } from 'visualogic';
// ...
context: ContextAwsApi & VisualogicContext
```

## error handle patterns

### [REUSE] `HelpfulError` for wrap

wrap sdk errors with context via `HelpfulError` [5][6][7].

```ts
// src/domain.operations/organization/getOneOrganization.ts:87-95
throw new HelpfulError('aws.getOneOrganization error', {
  cause: error,
  context: {
    errorName: error.name,
    errorMessage: error.message,
    input,
  },
});
```

### [REUSE] `BadRequestError` for validation

use `BadRequestError.throw` for input validation failures [5][6][7].

```ts
BadRequestError.throw('ref is neither unique nor primary', { input });
```

### [REUSE] idempotent error handle

handle "already exists" or "not found" errors idiomatically [6][7].

```ts
// src/domain.operations/organization/delOrganization.ts:91-94
if (error.name === 'AWSOrganizationsNotInUseException') {
  return { deleted: true };
}
```

## sdk client patterns

### [REUSE] client instantiation

create sdk client with region from context [5][6][7].

```ts
const client = new OrganizationsClient({
  region: context.aws.credentials.region,
});
```

### [EXTEND] IAMClient for account alias

account alias operations will use IAMClient (not OrganizationsClient) [10].

```ts
import { IAMClient, ListAccountAliasesCommand } from '@aws-sdk/client-iam';

const client = new IAMClient({
  region: context.aws.credentials.region,
});
```

note: iam is global service, but sdk still requires region parameter.
