# blueprint: account alias support

## summary

implement `DeclaredAwsAccountAlias` domain object with `getOneAccountAlias`, `setAccountAlias`, and `delAccountAlias` operations.

key design decisions:
- **singleton pattern**: one alias per account [R:access/3,5,11], retrieved via `by: { auth: true }` [R:prod/5]
- **no account reference**: aws api constraint — you can only manage your own account's alias, so account reference would be meaningless [R:vision/decided-questions]. note: this supersedes any earlier draft shapes that showed `account` field — the final vision decision (line 189-193) explicitly decided against it.
- **upsert semantics**: setAccountAlias uses upsert (not findsert) per rebrand use case [R:repros/journey2]
- **delete-then-create**: aws has no update api, so upsert = delete old + create new [R:access/idempotency]
- **validation first**: validate alias format before any aws api call [R:premortem/validation-mismatch]
- **partition uniqueness**: alias must be unique across all aws accounts in partition [R:access/1,8] — documented in error messages [R:premortem/partition-collision]
- **del returns confirmation**: `delAccountAlias` returns `{ deleted: true }` (not void) to match the idempotent delete pattern in `delOrganization.ts:56-58` [R:prod/7] and provide explicit confirmation

**criteria alignment notes**:
- domain object shape follows vision decided-questions (no account reference) over earlier draft shapes
- operation names use `getOneAccountAlias` per codebase convention (`getOneOrganization` pattern) over generic `getAccountAlias`
- del return type is `{ deleted: true }` per extant pattern, not void

### journey structure

the acceptance test (`src/contract/sdks/accountAlias.acceptance.test.ts`) exercises complete lifecycles:

```ts
// cleanup helper: attempts deletion, logs outcome
// note: cleanup is best-effort — beforeAll tolerates failure (alias may not exist),
// afterAll logs failure but doesn't block test completion
const cleanupAlias = async () => {
  await delAccountAlias({ by: { auth: true } }, context);
};

// beforeAll: tolerate failure (alias may not exist yet)
beforeAll(async () => cleanupAlias().catch(() => {}));
// afterAll: log failure but don't block test completion
afterAll(async () => cleanupAlias().catch(e => context.log.debug('post-test cleanup skipped', { e })));

given('[case1] complete account alias lifecycle', () => {
  // t0: verify clean state
  when('[t0] getOneAccountAlias is called on clean account', () => {
    const result = useThen('it returns null', async () =>
      getOneAccountAlias({ by: { auth: true } }, context)
    );
    then('it should return null', () => {
      expect(result).toBeNull();
      expect(result).toMatchSnapshot();
    });
  });

  // t1: create alias
  when('[t1] setAccountAlias is called to create', () => {
    const result = useThen('it creates the alias', async () =>
      setAccountAlias({ upsert: { alias: 'declastruct-test-alias' } }, context)
    );
    then('it should return the created alias', () => {
      expect(result.alias).toEqual('declastruct-test-alias');
      expect(result).toMatchSnapshot();
    });
  });

  // t2: verify alias persisted
  when('[t2] getOneAccountAlias is called after create', () => {
    const result = useThen('it returns the alias', async () =>
      getOneAccountAlias({ by: { auth: true } }, context)
    );
    then('it should return the alias', () => {
      expect(result?.alias).toEqual('declastruct-test-alias');
      expect(result).toMatchSnapshot();
    });
  });

  // t3: idempotent set
  when('[t3] setAccountAlias is called with same alias', () => {
    const result = useThen('it returns same alias', async () =>
      setAccountAlias({ upsert: { alias: 'declastruct-test-alias' } }, context)
    );
    then('it should return the same alias (idempotent)', () => {
      expect(result.alias).toEqual('declastruct-test-alias');
      expect(result).toMatchSnapshot();
    });
  });

  // t4: delete alias
  when('[t4] delAccountAlias is called', () => {
    const result = useThen('it deletes', async () =>
      delAccountAlias({ by: { auth: true } }, context)
    );
    then('it should return deleted: true', () => {
      expect(result).toEqual({ deleted: true });
      expect(result).toMatchSnapshot();
    });
  });

  // t5: verify deletion
  when('[t5] getOneAccountAlias is called after delete', () => {
    const result = useThen('it returns null', async () =>
      getOneAccountAlias({ by: { auth: true } }, context)
    );
    then('it should return null', () => {
      expect(result).toBeNull();
      expect(result).toMatchSnapshot();
    });
  });

  // t6: idempotent delete
  when('[t6] delAccountAlias is called again (idempotent)', () => {
    const result = useThen('it returns deleted again', async () =>
      delAccountAlias({ by: { auth: true } }, context)
    );
    then('it should return deleted: true without error', () => {
      expect(result).toEqual({ deleted: true });
      expect(result).toMatchSnapshot();
    });
  });
});

given('[case2] alias rebrand (update to different alias)', () => {
  // setup: ensure alias exists with old value
  beforeAll(async () => {
    await setAccountAlias({ upsert: { alias: 'declastruct-test-old' } }, context);
  });
  // cleanup: remove alias after case2 to not affect other tests
  afterAll(async () => cleanupAlias().catch(e => context.log.debug('case2 cleanup skipped', { e })));

  when('[t0] getOneAccountAlias is called', () => {
    const result = useThen('it returns old alias', async () =>
      getOneAccountAlias({ by: { auth: true } }, context)
    );
    then('it should return the old alias', () => {
      expect(result?.alias).toEqual('declastruct-test-old');
      expect(result).toMatchSnapshot();
    });
  });

  when('[t1] setAccountAlias is called with different alias', () => {
    const response = useThen('it returns the new alias', async () =>
      setAccountAlias({ upsert: { alias: 'declastruct-test-new' } }, context)
    );
    then('it should return the new alias', () => {
      expect(response.alias).toEqual('declastruct-test-new');
      expect(response).toMatchSnapshot();
    });
  });

  when('[t2] getOneAccountAlias is called after rebrand', () => {
    const result = useThen('it returns new alias', async () =>
      getOneAccountAlias({ by: { auth: true } }, context)
    );
    then('it should return the new alias', () => {
      expect(result?.alias).toEqual('declastruct-test-new');
      expect(result).toMatchSnapshot();
    });
  });
});

given('[case3] validation edge cases', () => {
  when('[t0] setAccountAlias is called with uppercase alias', () => {
    then('it should throw BadRequestError with constraint details', async () => {
      const error = await getError(
        setAccountAlias({ upsert: { alias: 'UPPERCASE' } }, context)
      );
      expect(error.name).toEqual('BadRequestError');
      expect(error.message).toContain('invalid account alias format');
      expect(error).toMatchSnapshot();
    });
  });
});
```

## risk mitigations

### partition collision [R:premortem/partition-collision]

if alias already taken by another account, CreateAccountAliasCommand throws EntityAlreadyExistsException [R:access/16].

**mitigation**: catch exception, throw BadRequestError with clear message:
```
alias 'commonly-used-name' already in use in partition
- AWS account aliases must be unique across all accounts in your partition (aws, aws-cn, aws-us-gov)
- try a namespaced alias like 'yourorg-commonly-used-name'
```

### delete-then-create atomicity [R:premortem/upsert-non-atomic]

upsert deletes old alias before create new. if create fails after delete, account has no alias.

**risk analysis**: while premortem rated this high severity, deeper analysis shows rollback is not viable:
- **transient failures** (network, throttle): user retries, create succeeds on second attempt
- **validation failures**: caught by validateAccountAliasFormat before delete — never reaches this state
- **partition collision**: alias taken in window — user chooses different alias
- **permanent failures** (aws outage): rollback would also fail

**why NOT rollback**:
1. rollback can also fail — if create failed due to AWS issue, recreate of old alias would likely fail too
2. rollback on collision is impossible — if new alias taken, old alias might also be taken by then
3. rollback adds complexity that's untestable in integration (requires simulated mid-operation failure)
4. the "recovery" path is identical: user calls setAccountAlias again with desired alias

**mitigation adopted**: document edge case in operation comments. log delete success to aid debug. if create fails, account is in "clean" state (no alias) which is safe — user retries with any desired alias.

### validation before aws call [R:premortem/validation-mismatch]

invalid alias format should fail fast with clear error, not confuse user with aws api errors.

**mitigation**: validateAccountAliasFormat runs before any aws call. BadRequestError lists all constraints for user to check.
