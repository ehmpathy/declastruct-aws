## constraint: IAM permissions

the demo-agent role lacks `iam:CreateAccountAlias` permission.

tests that require this permission now **fail loud** with clear error messages (not skipped):
- error: "ConstraintError: IAM CreateAccountAlias permission required for this test. Grant iam:CreateAccountAlias to the test role to run write tests."
- full error chain preserved for diagnosis

this satisfies:
- rule.forbid.failhide (no silent `.catch(() => {})`)
- rule.require.failfast (tests fail immediately with clear guidance)
- rule.require.failloud (errors include actionable hints with ConstraintError class)

to enable these tests, add to demo-agent role policy:
```json
{
  "Effect": "Allow",
  "Action": [
    "iam:CreateAccountAlias",
    "iam:DeleteAccountAlias",
    "iam:ListAccountAliases"
  ],
  "Resource": "*"
}
```
