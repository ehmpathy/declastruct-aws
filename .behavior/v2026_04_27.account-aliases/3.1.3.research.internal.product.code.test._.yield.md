# research: internal test code patterns

## summary

analysis of test codepath patterns in declastruct-aws relevant to account alias implementation.

## test infrastructure patterns

### [REUSE] dual context fixtures

two context fixtures for different test types [1][2].

| fixture | use case | credentials | async |
|---------|----------|-------------|-------|
| `getMockedAwsApiContext()` | unit tests | fake (`123456789012`) | sync |
| `getSampleAwsApiContext()` | integration tests | real (via `getCredentials()`) | async |

```ts
// src/.test/getMockedAwsApiContext.ts:9-12
export const getMockedAwsApiContext = (input?: {
  cacheDir?: string;
}): ContextAwsApi & ContextLogTrail => ({
  aws: {
    credentials: {
      region: 'us-east-1',
      account: '123456789012',
    },
    // ...
  },
});
```

```ts
// src/.test/getSampleAwsApiContext.ts:10-13
export const getSampleAwsApiContext = async (input?: {
  cacheDir?: string;
}): Promise<ContextAwsApi & ContextLogTrail> => ({
  aws: {
    credentials: await getCredentials(),
    // ...
  },
});
```

### [REUSE] test-fns `given/then` pattern

all tests use `given/then` from `test-fns` for BDD style [3][4][5][6].

```ts
// src/domain.operations/organization/getOneOrganization.test.ts:27-53
given('an organization exists', () => {
  then('we should return the casted organization', async () => {
    // ...
  });
});
```

### [REUSE] `useBeforeAll` for context setup

integration tests use `useBeforeAll` to create context once per suite [6][7].

```ts
// src/domain.operations/iamRole/getIamRole.integration.test.ts:7-8
const context = useBeforeAll(() => getSampleAwsApiContext());
```

## unit test patterns

### [REUSE] `jest.mock()` for SDK client

unit tests mock the entire SDK client module [3][4][5].

```ts
// src/domain.operations/organization/getOneOrganization.test.ts:12-18
jest.mock('@aws-sdk/client-organizations');

const mockSend = jest.fn();
(OrganizationsClient as jest.Mock).mockImplementation(() => ({
  send: mockSend,
}));
```

### [REUSE] mock collocated cast functions

unit tests mock cast functions to isolate operation logic [3][4].

```ts
// src/domain.operations/organization/getOneOrganization.test.ts:9-13
import * as castModule from './castIntoDeclaredAwsOrganization';
// ...
jest.mock('./castIntoDeclaredAwsOrganization');
// ...
(castModule.castIntoDeclaredAwsOrganization as jest.Mock).mockReturnValue({
  id: 'o-abc123xyz789',
  // ...
});
```

### [REUSE] mock collocated get operations for set/del

set and del tests mock getOne operations to control found state [4][5].

```ts
// src/domain.operations/organization/setOrganization.test.ts:9-15
import * as getOneModule from './getOneOrganization';
// ...
jest.mock('./getOneOrganization');
// ...
(getOneModule.getOneOrganization as jest.Mock).mockResolvedValue(foundBefore);
```

### [REUSE] `beforeEach` with `jest.clearAllMocks()`

unit tests clear mocks between cases [3][4][5].

```ts
// src/domain.operations/organization/getOneOrganization.test.ts:24-26
beforeEach(() => {
  jest.clearAllMocks();
});
```

### [REUSE] `expect.any(Command)` for command assertions

unit tests assert correct command types without exact input match [3][4][5].

```ts
// src/domain.operations/organization/getOneOrganization.test.ts:48-50
expect(mockSend).toHaveBeenCalledWith(
  expect.any(DescribeOrganizationCommand),
);
```

### [REUSE] error simulation via `mockRejectedValue`

unit tests simulate AWS errors by name [3][4][5].

```ts
// src/domain.operations/organization/getOneOrganization.test.ts:57-59
const error = new Error('Organization not in use');
error.name = 'AWSOrganizationsNotInUseException';
mockSend.mockRejectedValue(error);
```

## integration test patterns

### [REUSE] real AWS resources

integration tests hit real AWS API with real credentials [6][7][8].

```ts
// src/domain.operations/iamRole/getIamRole.integration.test.ts:10-31
given('a common aws service role', () => {
  then('we should be able to get the AWSServiceRoleForSupport role', async () => {
    const role = await getIamRole(
      { by: { unique: { name: 'AWSServiceRoleForSupport' } } },
      context,
    );
    expect(role).not.toBeNull();
    expect(role?.name).toBe('AWSServiceRoleForSupport');
  });
});
```

### [REUSE] test with known resources

integration tests use well-known AWS resources that exist in any account [6].

examples:
- `AWSServiceRoleForSupport` - exists in all accounts
- `AWSServiceRoleForTrustedAdvisor` - exists in all accounts

### [REUSE] idempotency verification

integration tests verify findsert/upsert idempotency [7].

```ts
// src/domain.operations/iamRole/setIamRole.integration.test.ts:49-53
then('findsert should be idempotent', async () => {
  const roleAgain = await setIamRole({ findsert: roleDesired }, context);
  expect(roleAgain.name).toBe(testRoleName);
});
```

### [REUSE] chain: set → get → verify

integration tests verify persistence via get after set [7].

```ts
// src/domain.operations/iamRole/setIamRole.integration.test.ts:39-47
then('we should be able to get the role we created', async () => {
  const role = await getIamRole(
    { by: { unique: { name: testRoleName } } },
    context,
  );
  expect(role).not.toBeNull();
  expect(role?.name).toBe(testRoleName);
});
```

### [REUSE] console.log for debug artifacts

integration tests log results for manual inspection [6][7].

```ts
// src/domain.operations/iamRole/setIamRole.integration.test.ts:36
console.log(roleAfter);
```

## test case patterns

### [REUSE] null return for not found

tests verify null return when resource absent [3][6].

```ts
// src/domain.operations/iamRole/getIamRole.integration.test.ts:34-48
given('a role that does not exist', () => {
  then('we should get null', async () => {
    const role = await getIamRole(
      { by: { unique: { name: 'declastruct-nonexistent-role-12345' } } },
      context,
    );
    expect(role).toBeNull();
  });
});
```

### [REUSE] idempotent delete (success when absent)

del operation tests verify idempotent success [5].

```ts
// src/domain.operations/organization/delOrganization.test.ts:72-80
given('no organization exists', () => {
  then('we should return success (idempotent)', async () => {
    (getOneModule.getOneOrganization as jest.Mock).mockResolvedValue(null);
    const result = await delOrganization({ by: sampleOrgRef }, context);
    expect(mockSend).not.toHaveBeenCalled();
    expect(result).toEqual({ deleted: true });
  });
});
```

### [REUSE] error wrap verification

tests verify HelpfulError wrap for unexpected errors [3][4][5].

```ts
// src/domain.operations/organization/getOneOrganization.test.ts:82-92
given('an unexpected error occurs', () => {
  then('we should throw a HelpfulError', async () => {
    const error = new Error('Some unexpected error');
    error.name = 'InternalServerError';
    mockSend.mockRejectedValue(error);
    await expect(
      getOneOrganization({ by: { auth: true } }, context),
    ).rejects.toThrow('aws.getOneOrganization error');
  });
});
```

### [REUSE] validation error tests

tests verify BadRequestError for invalid input [4][5].

```ts
// src/domain.operations/organization/setOrganization.test.ts:182-196
given('managementAccount mismatch before creation', () => {
  then('we should throw a BadRequestError', async () => {
    await expect(
      setOrganization({ findsert: desiredOrgWithWrongAccount }, context),
    ).rejects.toThrow('managementAccount mismatch');
  });
});
```

## patterns specific to account alias

### [EXTEND] IAMClient mock setup

account alias tests will mock IAMClient instead of OrganizationsClient.

```ts
// pattern for unit tests
import { IAMClient, ListAccountAliasesCommand } from '@aws-sdk/client-iam';

jest.mock('@aws-sdk/client-iam');

const mockSend = jest.fn();
(IAMClient as jest.Mock).mockImplementation(() => ({
  send: mockSend,
}));
```

### [EXTEND] singleton `by: { auth: true }` test cases

account alias uses singleton pattern like organization [3][4][5].

```ts
// pattern: test getAccountAlias with by: { auth: true }
given('an alias exists', () => {
  then('we should return the alias', async () => {
    const result = await getAccountAlias({ by: { auth: true } }, context);
    expect(result).not.toBeNull();
    expect(result?.alias).toBe('ahbode-prep');
  });
});
```

### [EXTEND] upsert via delete-then-create test

account alias set requires delete + create pattern (no update API).

```ts
// pattern: test setAccountAlias upsert behavior
given('an alias already exists', () => {
  then('upsert should delete old and create new', async () => {
    mockSend.mockImplementation((command) => {
      if (command instanceof DeleteAccountAliasCommand) return {};
      if (command instanceof CreateAccountAliasCommand) return {};
    });
    // ...
  });
});
```

### [EXTEND] alias validation error test

test alias format validation before AWS call.

```ts
// pattern: test setAccountAlias validation
given('invalid alias format', () => {
  then('we should throw BadRequestError before AWS call', async () => {
    await expect(
      setAccountAlias({ upsert: { alias: 'INVALID' } }, context),
    ).rejects.toThrow('invalid alias format');
    expect(mockSend).not.toHaveBeenCalled();
  });
});
```

### [EXTEND] integration test with real alias

integration tests will set/get/del real alias on test account.

note: partition-unique constraint means test alias must be unique (e.g., `declastruct-test-{timestamp}`).
