# research: external product access

## summary

account alias management requires access to AWS IAM API via `@aws-sdk/client-iam`. three commands are needed: `ListAccountAliasesCommand`, `CreateAccountAliasCommand`, `DeleteAccountAliasCommand`.

## remote repositories

### AWS IAM API

- **endpoint**: `iam.amazonaws.com` (global service)
- **sdk**: `@aws-sdk/client-iam`
- **auth**: IAM credentials with appropriate permissions

## contracts

### CreateAccountAliasCommand

| attribute | value |
|-----------|-------|
| input | `{ AccountAlias: string }` |
| output | `{}` (empty on success) |
| errors | `EntityAlreadyExistsException`, `ConcurrentModificationException`, `LimitExceededException`, `ServiceFailureException` |

alias constraints [1][4][10]:
- 3-63 characters
- lowercase letters, digits, hyphens
- pattern: `^[a-z0-9]([a-z0-9]|-(?!-)){1,61}[a-z0-9]$`
- no consecutive hyphens
- must start/end with letter or digit

### ListAccountAliasesCommand

| attribute | value |
|-----------|-------|
| input | `{ Marker?: string, MaxItems?: number }` |
| output | `{ AccountAliases: string[], IsTruncated: boolean, Marker?: string }` |
| errors | `ServiceFailureException` |

note: aws supports only one alias per account [3][5][11]

### DeleteAccountAliasCommand

| attribute | value |
|-----------|-------|
| input | `{ AccountAlias: string }` |
| output | `{}` (empty on success) |
| errors | `NoSuchEntityException`, `ConcurrentModificationException`, `LimitExceededException`, `ServiceFailureException` |

## iam permissions required

```json
{
  "Effect": "Allow",
  "Action": [
    "iam:CreateAccountAlias",
    "iam:DeleteAccountAlias",
    "iam:ListAccountAliases"
  ],
  "Resource": "*"
}
```

note: these actions do not support resource-level permissions — must use `"Resource": "*"` [7]

## sdk usage patterns

### client instantiation [4][9][15]

```ts
import { IAMClient } from "@aws-sdk/client-iam";

const client = new IAMClient({});  // uses default credential chain
```

### command execution [4][5][6]

```ts
import {
  IAMClient,
  CreateAccountAliasCommand,
  ListAccountAliasesCommand,
  DeleteAccountAliasCommand
} from "@aws-sdk/client-iam";

const client = new IAMClient({});

// list
const { AccountAliases } = await client.send(new ListAccountAliasesCommand({}));

// create
await client.send(new CreateAccountAliasCommand({ AccountAlias: "ahbode-prep" }));

// delete
await client.send(new DeleteAccountAliasCommand({ AccountAlias: "ahbode-prep" }));
```

### error handle patterns [16]

```ts
import { EntityAlreadyExistsException, NoSuchEntityException } from "@aws-sdk/client-iam";

try {
  await client.send(new CreateAccountAliasCommand({ AccountAlias: alias }));
} catch (error) {
  if (error instanceof EntityAlreadyExistsException) {
    // alias taken by another account
  }
  throw error;
}
```

## best practices

### credential management [13][14]

- use default credential chain (sdk auto-discovers credentials)
- prefer temporary credentials via IAM roles
- grant least privilege permissions

### idempotency considerations

- `CreateAccountAlias` throws if alias already exists — check first via `ListAccountAliases`
- `DeleteAccountAlias` throws `NoSuchEntityException` if alias absent — check first via `ListAccountAliases`
- upsert pattern: list → delete if different → create [1][2]

### sdk v3 patterns [9][15]

- use modular imports (tree-shake reduces bundle size from ~40MB to <100KB)
- use async/await with `.send()` method
- each operation maps to a dedicated Command class

## convergence signals

multiple sources confirm:
- **one alias per account** — sources [3][5][8][11] all state this explicitly
- **partition-unique** — sources [1][8] confirm uniqueness within aws partition
- **3-63 char constraint** — sources [1][2][3][4][10][11][12] all agree

## anti-patterns

### from sources [13][14]

- **avoid hardcoded credentials** — let sdk use credential chain
- **avoid over-provision of permissions** — grant only iam:*AccountAlias actions needed
- **avoid silent error swallow** — always handle EntityAlreadyExistsException for create, NoSuchEntityException for delete

### from declastruct codebase patterns

- **avoid raw sdk calls in orchestrators** — wrap in communicators
- **avoid upsert without get-first pattern** — check state before mutate
