# blackbox criteria: account alias

## usecase.1 = get account alias

### happy path

given an account with an alias set
  when getAccountAlias is called with by.auth
    then it returns the alias with account reference
      sothat users can verify the current alias

given an account with no alias set
  when getAccountAlias is called with by.auth
    then it returns null
      sothat users know no alias exists

## usecase.2 = set account alias

### happy path — first alias

given an account with no alias
  when setAccountAlias is called with a valid alias
    then the alias is created
    then the response includes the account and alias
      sothat users can confirm the alias was set

### happy path — update alias

given an account with an alias
  when setAccountAlias is called with a different valid alias
    then the old alias is replaced with the new alias
    then the response includes the account and new alias
      sothat users can rebrand their account

### idempotency

given an account with alias "ahbode-prep"
  when setAccountAlias is called with alias "ahbode-prep"
    then the alias remains "ahbode-prep"
    then no error is raised
      sothat repeated calls are safe

### error — alias taken by another account

given another account in the partition already uses alias "ahbode-prep"
  when setAccountAlias is called with alias "ahbode-prep"
    then an error is raised with clear message about partition uniqueness
      sothat users understand why the alias failed
      sothat users know to choose a different alias

### error — invalid alias format

given an alias with invalid characters (e.g., uppercase, underscore)
  when setAccountAlias is called
    then an error is raised before the AWS call
      sothat users get fast feedback on format issues

given an alias shorter than 3 characters
  when setAccountAlias is called
    then an error is raised with minimum length constraint
      sothat users know the length requirements

given an alias longer than 63 characters
  when setAccountAlias is called
    then an error is raised with maximum length constraint
      sothat users know the length requirements

given an alias with consecutive hyphens
  when setAccountAlias is called
    then an error is raised about consecutive hyphens
      sothat users understand the constraint

given an alias that starts or ends with a hyphen
  when setAccountAlias is called
    then an error is raised about start/end character requirements
      sothat users know aliases must start/end with letter or digit

## usecase.3 = delete account alias

### happy path

given an account with an alias
  when delAccountAlias is called with by.auth
    then the alias is removed
      sothat the account reverts to account-id-only display

### idempotency

given an account with no alias
  when delAccountAlias is called with by.auth
    then no error is raised
      sothat repeated calls are safe
