# blueprint criteria: account alias

## blackbox criteria satisfied

- usecase.1 = get account alias ✓
- usecase.2 = set account alias ✓
- usecase.3 = delete account alias ✓

## subcomponent contracts

### domain object

given DeclaredAwsAccountAlias contract
  then declares account: RefByPrimary<typeof DeclaredAwsOrganizationAccount>
  then declares alias: string
  then static primary = ['alias']
  then static unique = ['account']

### operations

given getAccountAlias contract
  then exposes: getAccountAlias({ by: { auth: true } }, context) => DeclaredAwsAccountAlias | null
  then returns null when no alias is set

given setAccountAlias contract
  then exposes: setAccountAlias({ upsert: { alias: string } }, context) => DeclaredAwsAccountAlias
  then validates alias format before AWS call
  then throws BadRequestError for invalid alias format
  then throws error with clear message when alias is taken in partition

given delAccountAlias contract
  then exposes: delAccountAlias({ by: { auth: true } }, context) => void
  then is idempotent (no error when alias absent)

## composition boundaries

given account alias operations
  then all operations use IAMClient from @aws-sdk/client-iam
  then all operations derive account from auth context
  then setAccountAlias composes: validate → get current → delete if different → create
  then delAccountAlias composes: get current → delete if present
