# 5.3 verification checklist

wish: add exclusion support to `DeclaredAwsIamPolicyStatement` so iam statements express
the aws "deny all except X" idiom, mapped to `NotResource` / `NotAction` / `NotPrincipal`.

final design (pivoted from the initial `{ except }` sketch): every selector field is
`bare | Scope`, where `Scope = { include?, exclude? }`. `include` maps to the positive aws
element, `exclude` maps to the `Not*` element, and a bare scalar/array/principal is shorthand
for `{ include: X }`. two wrapper domain objects carry the scope: `DeclaredAwsIamStatementScope`
(string payload, for resource/action) and `DeclaredAwsIamPrincipalScope` (principal payload).

this is an enhancement to an extant resource (iam policy statement, embedded in iam role
inline policies) — not a new resource type. no new cli/sdk contract surface; the caller
experience is the declarative `{ include, exclude }` shape on domain fields.

## upstream dependency

the always-object `Scope` design required two domain-objects fixes (landed in 0.32.2):
1. element-level bare-value skip — bare scalars inside arrays under a nested key are not hydrated
2. structural multi-option nested disambiguation — a multi-option nested settles by strict schema
   fit without a `_dobj` discriminator

both participating principal classes therefore declare strict (closed) zod schemas so the
structural disambiguation is unambiguous. see the future-traveler brief:
`.agent/repo=.this/role=any/briefs/howto.domain-objects-nested-unions.md`.

## idempotency fixes (required to reach a stable acceptance run)

the acceptance suite bundles all resource types; prior non-idempotent vpc/ec2 set operations
(from PRs #51/#54) blocked a clean run and were fixed fundamentally:

| operation | defect | fix |
|-----------|--------|-----|
| setVpcSubnet | `InvalidSubnet.Conflict` on a cidr held by an untagged orphan | adopt-on-conflict: find orphan by (vpc, cidr), tag with our exid |
| setVpcRouteTable | `CreateRoute` re-run throws `RouteAlreadyExists` | tolerate the extant-conflict as idempotent success |
| setVpcRouteTable | subnet association never reconciled (assoc held by another table) | new `setVpcRouteTableAssociations`: skip-if-ours, move-if-elsewhere, drop-if-undesired |
| setVpcInternetGateway | `AttachInternetGateway` re-run throws `Resource.AlreadyAssociated` | tolerate the extant-conflict |
| setVpcSecurityGroup | `AuthorizeSecurityGroup*` re-run throws `InvalidPermission.Duplicate` | tolerate the extant-conflict |

shared helper: `src/domain.operations/aws/tolerateExtantConflict.ts` (allowlisted error names only;
all other errors rethrow, so failfast is preserved).
