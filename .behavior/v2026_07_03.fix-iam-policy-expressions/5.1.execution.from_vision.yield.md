# 5.1 execution — iam policy statement exclusion support

execute the decided contract from `1.vision.yield.md`: let `DeclaredAwsIamPolicyStatement`
express AWS's "deny all except X" idiom, serialized to `NotResource`/`NotAction`/`NotPrincipal`,
round-tripped on read, with fail-fast on malformed input.

## design decision settled at execution: `{ include, exclude }` Scope

the initial sketch was a `{ except }` variant. it was replaced (with human direction) by an
always-object `Scope`:

- every selector field is `bare | Scope`, where `Scope = { include?, exclude? }`
- `include` → positive AWS element; `exclude` → `Not*` element; bare scalar/array/principal is
  shorthand for `{ include: X }`
- two wrapper domain objects carry the scope:
  - `DeclaredAwsIamStatementScope` — string payload (resource, action)
  - `DeclaredAwsIamPrincipalScope` — principal payload
- **zero constructor overrides** (hard constraint). the fields hydrate through `static nested`;
  `principal` is a multi-option nested `[DeclaredAwsIamPrincipal, DeclaredAwsIamPrincipalScope]`
  that settles by strict zod schema fit.

### upstream dependency (landed 0.32.2)

the always-object design needed two domain-objects fixes, both dispatched + landed:
1. element-level bare-value skip — bare scalars in arrays under a nested key are not hydrated
2. structural multi-option nested disambiguation — settles by strict schema fit, no `_dobj`

both principal classes therefore declare strict (closed) zod schemas. captured for future
travelers in `.agent/repo=.this/role=any/briefs/howto.domain-objects-nested-unions.md`.
