# blueprint: verify SSH connection & hibernate capacity

## summary

build verification infrastructure for EC2 SSH connection and hibernation via:

1. **domain objects** — `DeclaredAwsEc2SshKeyAuthorized`, `DeclaredAwsSsmSshTunnel`
2. **operations** — get/set for SSH keys and SSH tunnels
3. **integration tests** — SSM connectivity, SSH key authorization, SSH tunnel lifecycle, hibernation state transitions
4. **acceptance tests** — EC2 resources in declastruct CLI workflow (plan/apply/idempotent)
5. **infrastructure** — ephemeral NAT instance (fck-nat AMI) for SSM connectivity in acceptance env, IAM roles

### terminology

| term | definition |
|------|------------|
| **gen** | find-or-create: idempotent pattern (gen* verbs) that returns extant record or creates new |
| **fck-nat** | lightweight NAT gateway AMI for AWS (cost-effective NAT instance) |
| **yield** | research document produced by prior route milestone (e.g., `code.prod._.yield.md`) |
| **exid** | external identifier: unique key from external system (e.g., AWS instance ID) |
| **bastion** | jump host: EC2 instance that provides SSH access to private subnet resources |
| **self-usage** | use of our own product internally (validates real-world patterns) |

### coverage by case (SSH key authorization)

| case | what it covers |
|------|----------------|
| positive: instance active, key not authorized | key added to authorized_keys |
| positive: instance active, key already authorized | no-op, idempotent |
| negative: instance stopped, key not authorized | error: instance must be active |
| edge: stopped + already authorized | verified via SSM Parameter Store cache |

### coverage by case (SSH tunnel)

| case | what it covers |
|------|----------------|
| positive: tunnel OPEN | tunnel opened, port forwards |
| positive: tunnel already OPEN | no-op, idempotent |
| positive: tunnel CLOSED | tunnel terminated, port released |
| edge: tunnel already CLOSED | no-op, idempotent |
| negative: instance stopped | error: instance must be active |
| negative: SSM unreachable | error: SSM agent not reachable |

### coverage by case (hibernation)

| case | what it covers |
|------|----------------|
| positive: active → hibernated | hibernates, charges stop |
| positive: hibernated → active | resumes, SSM reachable |
| positive: already hibernated | no-op, idempotent |
| positive: already active | no-op, idempotent |
| negative: hibernation not enabled | error surfaced |
