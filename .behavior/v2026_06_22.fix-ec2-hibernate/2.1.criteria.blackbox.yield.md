# blackbox criteria: EC2 SSH & hibernate

## usecase.1 = authorize SSH key on EC2 instance

```
given([case1] EC2 instance is active)
  when(user declares SSH key authorized and applies)
    then(key is added to instance authorized_keys)
      sothat(user can authenticate via that key)
    then(key is cached for offline lookup)
      sothat(declastruct can check authorization when instance is stopped)

given([case2] EC2 instance is stopped)
  when(user declares SSH key authorized and applies)
    then(error: instance must be active to authorize keys)
      sothat(user knows to start instance first)

given([case3] SSH key already authorized)
  when(user re-applies same key declaration)
    then(no changes made)
      sothat(operation is idempotent)
```

## usecase.2 = open SSH tunnel via SSM

```
given([case1] SSH key is authorized on active instance)
  when(user declares SSH tunnel with status 'OPEN' and applies)
    then(local port forwards to instance SSH port)
      sothat(user can SSH to localhost:port)
    then(tunnel process runs in background)
      sothat(user can work in terminal)

given([case2] tunnel already open on same port)
  when(user re-applies same tunnel declaration)
    then(no new tunnel created)
      sothat(operation is idempotent)

given([case3] tunnel is open)
  when(user declares status 'CLOSED' and applies)
    then(tunnel process is terminated)
      sothat(port is released)
```

## usecase.3 = hibernate EC2 instance

```
given([case1] EC2 instance is active with hibernation enabled)
  when(user declares session status 'hibernated' and applies)
    then(instance enters hibernated state)
      sothat(RAM is preserved to EBS)
    then(compute charges stop)
      sothat(cost is reduced)

given([case2] instance already hibernated)
  when(user re-applies status 'hibernated')
    then(no changes made)
      sothat(operation is idempotent)

given([case3] instance without hibernation capability)
  when(user declares status 'hibernated' and applies)
    then(error: hibernation not supported)
      sothat(user knows constraint)
```

## usecase.4 = resume hibernated instance

```
given([case1] EC2 instance is hibernated)
  when(user declares session status 'active' and applies)
    then(instance resumes from hibernate)
      sothat(RAM state is restored)
    then(instance is reachable via SSM)
      sothat(user can establish sessions)

given([case2] instance already active)
  when(user re-applies status 'active')
    then(no changes made)
      sothat(operation is idempotent)
```

## usecase.5 = provision EC2 with hibernation

```
given([case1] no EC2 infrastructure exists)
  when(user declares launch template + instance + session and applies)
    then(launch template is created with hibernation config)
    then(instance is launched from template)
    then(instance reaches desired lifecycle status)
      sothat(full stack is provisioned declaratively)

given([case2] infrastructure already matches declaration)
  when(user generates plan)
    then(plan shows no changes)
      sothat(state convergence is verified)
```

## usecase.6 = verify SSM connectivity

```
given([case1] EC2 instance is active with IAM role)
  when(SSM command is sent to instance)
    then(command executes successfully)
    then(output is returned)
      sothat(connectivity is proven)

given([case2] instance lacks SSM agent or IAM role)
  when(SSM command is sent)
    then(error: SSM not reachable)
      sothat(user knows prerequisites are absent)
```
