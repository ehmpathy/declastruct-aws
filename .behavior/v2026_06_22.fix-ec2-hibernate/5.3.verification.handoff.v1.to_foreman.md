# 5.3.verification.handoff.v1.to_foreman

## blocker: AWS vCPU quota exhausted

### what i tried

1. **ran integration tests** — all 33 EC2-related tests fail with `VcpuLimitExceeded`
2. **analyzed the error** — AWS test account has a 16 vCPU limit, currently exhausted
3. **traced root cause** — extant EC2 instances (from prior test runs or other processes) consume the quota
4. **verified code is correct** — the tests use t2.micro/t3.micro (free-tier eligible), the issue is quota not instance type

### why each approach failed

| approach | why it failed |
|----------|---------------|
| run tests | VcpuLimitExceeded — quota at 16, fully consumed |
| use smaller instances | already use t2.micro (smallest) |
| wait for cleanup | instances persist — need manual termination |
| mock the tests | forbidden by `rule.forbid.integration.mocks` |
| skip the tests | forbidden by verification stone requirements |

### what makes this fundamentally impossible without foreman intervention

the AWS vCPU quota is an account-level limit controlled via AWS console. i cannot:
- terminate instances without AWS console access
- request quota increases without AWS console access
- bypass the limit programmatically

### is this truly a "foreman possesses the key" situation?

**yes.** the foreman has AWS console access to the test account. the mechanic does not.

### foreman action required

**option 1: terminate extant instances**
1. log into AWS console for the test account
2. navigate to EC2 > Instances
3. terminate instances that consume the vCPU quota
4. tests will then be able to create new instances

**option 2: request quota increase**
1. log into AWS console
2. navigate to Service Quotas > Amazon EC2
3. request increase for "On-Demand Standard instances" (vCPU limit)
4. current limit: 16 vCPUs, recommended: 32+ vCPUs

### after resolution

once the vCPU quota is freed, run:

```bash
rhx route.stone.set --stone 5.3.verification --as rewound
```

this will rewind the stone so verification can be re-attempted with the full test suite.

### rewind instruction

```bash
rhx route.stone.set --stone 5.3.verification --as rewound
```
