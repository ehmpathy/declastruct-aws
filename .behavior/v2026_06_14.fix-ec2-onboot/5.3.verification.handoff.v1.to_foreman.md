# handoff: verification guard configuration issue

## blocker

all 7 peer reviews fail with "combined scope resolves to zero files" due to guard configuration bug.

## the issue

the verification guard uses `--join intersect` with two incompatible patterns:

```yaml
--paths-with '**/*.ts' --paths-with '**/*.snap' --join intersect
```

intersect requires files to match BOTH patterns. no file is both `.ts` and `.snap`, so the intersection is always empty (0 files).

## what i tried

1. **read the guard file** — confirmed the pattern configuration
2. **checked each review** — all 7 fail with same "0 files" error
3. **analyzed the logic** — intersect is the wrong join for these patterns

## what needs to happen

change `--join intersect` to `--join union` in the verification guard peer reviews, OR use separate review invocations for `.ts` and `.snap` files.

## evidence
