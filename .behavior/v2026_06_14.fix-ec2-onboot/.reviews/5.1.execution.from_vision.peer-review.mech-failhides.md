# blocker.1: Error thrown without context metadata

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/pitofsuccess.errors/rule.require.failloud.md

**locations**:
- src/domain.operations/provider/getDeclastructAwsProvider.ts

Multiple errors are thrown using BadRequestError.throw and UnexpectedCodePathError.throw but without the required context/metadata object. Per the rule, errors must include context for immediate diagnosis, distinguishing caller vs server issues, and to avoid bare 'throw new Error' patterns. This is a blocker as it leads to hours of debug without actionable info.

**snippet**:
```ts
if (!region)
    BadRequestError.throw(
      'AWS region not specified. Set AWS_REGION, AWS_DEFAULT_REGION env var, or configure region in ~/.aws/config.',
    );

  // resolve account from STS identity
  const account = await getAccountFromSts();

  return { region, account };
};

const getAccountFromSts = async (): Promise<string> => {
  const sts = new STSClient({});
  const identity = await sts.send(new GetCallerIdentityCommand({}));
  return (
    identity.Account ??
    UnexpectedCodePathError.throw('failed to resolve AWS account id from STS')
  );
};
```
