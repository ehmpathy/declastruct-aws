# blocker.2: launch template optional support not implemented for instance creation

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.require.behavior-intent-coverage.md

**locations**:
- src/domain.operations/ec2Instance/setEc2Instance.ts

The vision explicitly answers the open question 'should launch template be required on instance? no' and declares template: Ref | null (optional for non-template instances and backwards compat). However, setEc2Instance explicitly throws if no launchTemplateId, stating 'cannot create instance without template' and only supports get for extant non-template instances. This prevents provisioning new instances without templates, failing to fulfill the optional template requirement in the 3-layer model.

**snippet**:
```ts
  // if no template, instance must already be extant (backwards compat)
  // cannot create new instances without template (need AMI and instance type)
  if (!launchTemplateId)
    return UnexpectedCodePathError.throw(
      'cannot create instance without template — template provides required AMI and instance type; for extant instances, ensure exid tag is set',
      { instance },
    );
```

# blocker.3: template reference not recovered on instance cast/get

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.require.behavior-intent-coverage.md

**locations**:
- src/domain.operations/ec2Instance/castIntoDeclaredAwsEc2Instance.ts

The vision defines a 3-layer model where DeclaredAwsEc2Instance references a launch template (template: Ref | null). However, castIntoDeclaredAwsEc2Instance always hardcodes template: null with the comment 'template ref not recoverable from AWS response'. This means get after apply will never return the template association for instances launched from templates, breaking roundtrip declaration and readback of the model. The AWS Instance type includes a LaunchTemplate field that could be used to resolve and set the ref.

**snippet**:
```ts
  return assure(
    DeclaredAwsEc2Instance.as({
      id: input.InstanceId,
      exid: exidTag.Value,
      template: null, // template ref not recoverable from AWS response
      subnetId: input.SubnetId ?? '',
      securityGroupIds,
      privateIp: input.PrivateIpAddress,
    }),
    hasReadonly({ of: DeclaredAwsEc2Instance }),
  );
```
