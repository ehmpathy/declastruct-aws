# blocker.1: Inline decode-friction in orchestrator for launch template ID resolution

**rule**: .agent/repo=ehmpathy/role=mechanic/briefs/practices/code.prod/readable.narrative/rule.forbid.inline-decode-friction.md

**locations**:
- src/domain.operations/ec2Instance/setEc2Instance.ts

The setEc2Instance function (an orchestrator composing gets and AWS send) contains a large inline async IIFE with multiple branches, ref type checks (isRefByPrimary), conditional return null, and a get call to resolve launchTemplateId. This requires readers to mentally simulate the logic and branches to understand the output. Per the rule, decode-friction (complex conditionals and ref routing) must be extracted to a named transformer so the orchestrator reads as pure narrative.

**snippet**:
```ts
  // lookup launch template id from ref
  const launchTemplateId = await (async (): Promise<string | null> => {
    // template=null means backwards compat with extant instances
    // (instances created outside declastruct, or before template support)
    // we cannot create new instances without a template because AMI and instance type are required
    if (!instance.template) return null;

    // if ref has id, use it directly
    if (isRefByPrimary({ of: DeclaredAwsEc2LaunchTemplate })(instance.template))
      return instance.template.id;

    // otherwise, look up template by unique key
    const template = await getEc2LaunchTemplate(
      {
        by: {
          ref: instance.template as Ref<typeof DeclaredAwsEc2LaunchTemplate>,
        },
      },
      context,
    );

    if (!template)
      return UnexpectedCodePathError.throw(
        'launch template not found; cannot create instance',
        { templateRef: instance.template },
      );

    return template.id;
  })();
```

    });
  })();
