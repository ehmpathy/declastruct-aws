# blocker.1: Non-idempotent mutation pattern and forbidden 'set' verb for EC2 session

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.maintenance-hazards.md

**locations**:
- src/domain.operations/ec2InstanceSession/setEc2InstanceSession.ts
- src/access/daos/DeclaredAwsEc2InstanceSessionDao.ts

The setEc2InstanceSession procedure and its DAO mapping violate idempotency requirements. It uses a non-standard input shape { session: ... } instead of PickOne<{findsert, upsert}>, and both findsert and upsert delegate to the identical implementation without distinguishing retry semantics. Function name uses forbidden 'set' verb (ambiguous for mutations per rules). This creates risk of duplicate side effects on retry and makes code hard to reason about for safe composition and retries. Must use only findsert/upsert/delete with corresponding input shapes for state-modifying procedures.

**snippet**:
```ts
  set: {
    findsert: async (input, context) => {
      return setEc2InstanceSession({ session: input }, context);
    },
    upsert: async (input, context) => {
      return setEc2InstanceSession({ session: input }, context);
    },
    delete: null, // sessions are not deleted; set to desired state instead
  },
});
```

# blocker.3: Complex nested conditionals violating narrative flow

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.maintenance-hazards.md

**locations**:
- src/domain.operations/vpcTunnel/setVpcTunnel.ts

setVpcTunnel contains multiple nested if blocks (portInUse > cacheFilePresent > process checks > health checks) plus callback-based promise setup for process events. This creates branched paths, deep indentation, and no single clear linear path, violating narrative flow requirements (early returns for guards, no nested branches, one clear path through logic). The tunnel lifecycle logic (idempotency checks, port reclamation, startup wait, cleanup) is hard to follow and will cause debug hours or silent defects on retries/races.

**snippet**:
```ts
  if (portInUse) {
    const cacheFilePresent = await isFilePresent({ path: cachePath });

    // port used by unknown process; kill it to reclaim
    if (!cacheFilePresent) {
      killProcessOnPort({ port: input.from.port });
    }

    // check if existing tunnel is ours and healthy
    if (cacheFilePresent) {
      const cacheContent = await fs.readFile(cachePath, 'utf-8');
      const cache: TunnelCacheFile = JSON.parse(cacheContent);

      // return early if existing tunnel is alive and healthy (idempotent)
      if (
        isProcessAlive({ pid: cache.pid }) &&
        (await isTunnelHealthy({ port: input.from.port }))
      )
        return castIntoDeclaredAwsVpcTunnel({
          unique: input,
          status: 'OPEN',
          pid: cache.pid,
        });

      // cleanup stale tunnel
      if (isProcessAlive({ pid: cache.pid }))
        process.kill(cache.pid, 'SIGTERM');
      await fs.rm(cachePath, { force: true });
    }
  }
```
