# blackbox criteria: fix vpc tunnel env differentiation

## usecase.1 = tunnel identity includes account + region

given a tunnel declaration with explicit account and region
  when declastruct plans the tunnel
    then the tunnel identity includes the account and region
      sothat tunnels for different accounts/regions are distinct resources
    then the same logical name in different accounts produces different tunnel identities
      sothat dev ahbodedb and prod ahbodedb are not confused

given a tunnel declaration without account field
  when declastruct validates the declaration
    then an error is surfaced
      sothat users know the account field is required

given a tunnel declaration without region field
  when declastruct validates the declaration
    then an error is surfaced
      sothat users know the region field is required

## usecase.2 = parallel tunnels for different accounts/regions

given a prod tunnel is already open (account 398838478359, region us-east-1, port 15433)
  when user runs tunnel declaration for dev (account 874711128849, region us-east-1, port 15432)
    then declastruct creates a new tunnel (not "in sync")
      sothat dev and prod tunnels coexist
    then the dev tunnel opens on port 15432
      sothat dev connections use dev resources

given a dev tunnel is already open (account 874711128849, region us-east-1, port 15432)
  when user runs tunnel declaration for prod (account 398838478359, region us-east-1, port 15433)
    then declastruct creates a new tunnel (not "in sync")
      sothat dev and prod tunnels coexist
    then the prod tunnel opens on port 15433
      sothat prod connections use prod resources

given a tunnel is open (account X, region us-east-1, port Y)
  when user runs tunnel declaration for same account but different region (account X, region us-west-2, port Z)
    then declastruct creates a new tunnel (not "in sync")
      sothat tunnels for different regions coexist

## usecase.3 = idempotent tunnel operations

given a tunnel declaration with account X, region R, port Y
  when the same declaration is run twice
    then the second run reports "in sync" (KEEP)
      sothat idempotent behavior is preserved
    then no new tunnel is opened
      sothat resources are not duplicated

given a tunnel declaration with account X, region R, port Y
  when a declaration with account Z, region R, port Y is run (same port, different account)
    then an error is surfaced about port collision
      sothat users know they cannot bind two tunnels to the same port

given a tunnel declaration with account X, region R1, port Y
  when a declaration with account X, region R2, port Y is run (same port, different region)
    then an error is surfaced about port collision
      sothat users know they cannot bind two tunnels to the same port
