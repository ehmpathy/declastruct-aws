# blackbox criteria matrix: fix vpc tunnel env differentiation

## matrix.1 = tunnel identity

dimensions:
- ind: account field presence (present, absent)
- ind: region field presence (present, absent)
- ind: account value (same, different)
- ind: region value (same, different)
- dep: identity outcome (distinct, same, error)

| ind: account present | ind: region present | ind: account values | ind: region values | dep: identity outcome |
|---------------------|---------------------|---------------------|--------------------|-----------------------|
| yes | yes | same | same | same identity |
| yes | yes | different | same | distinct identity |
| yes | yes | same | different | distinct identity |
| yes | yes | different | different | distinct identity |
| no | yes | n/a | any | error (account required) |
| yes | no | any | n/a | error (region required) |
| no | no | n/a | n/a | error (both required) |

## matrix.2 = parallel tunnel operations

dimensions:
- ind: tunnel A account
- ind: tunnel B account
- ind: tunnel A region
- ind: tunnel B region
- ind: tunnel A port
- ind: tunnel B port
- dep: coexistence outcome
- dep: tunnel B action

| ind: acct A | ind: acct B | ind: region A | ind: region B | ind: port A | ind: port B | dep: coexist | dep: action B |
|-------------|-------------|---------------|---------------|-------------|-------------|--------------|---------------|
| X | X | R | R | Y | Y | yes (same tunnel) | KEEP |
| X | Z | R | R | Y | Y | no (port collision) | ERROR |
| X | X | R | S | Y | Y | no (port collision) | ERROR |
| X | Z | R | S | Y | Y | no (port collision) | ERROR |
| X | Z | R | R | Y | W | yes | CREATE |
| X | X | R | S | Y | W | yes | CREATE |
| X | X | R | R | Y | W | yes | CREATE |

where:
- X, Z = different accounts
- R, S = different regions
- Y, W = different ports

## gap analysis

**no gaps found.** all meaningful combinations are covered:
1. same account + same region + same port = idempotent (KEEP)
2. diff account + same port = error (port collision)
3. diff region + same port = error (port collision)
4. diff account + diff port = coexist (CREATE)
5. diff region + diff port = coexist (CREATE)
6. same account + same region + diff port = create new (CREATE)

## decomposition assessment

**no decomposition needed.** the matrix has 4-6 dimensions per usecase, which is manageable. the behavior is focused on a single concern: account + region based tunnel identity.
