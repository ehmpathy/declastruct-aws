## pattern.3 = getTunnelHash account differentiation test

**source**: `src/domain.operations/vpcTunnel/utils/getTunnelHash.test.ts`

```ts
// [3] tests that different credentials produce different hashes
given('same tunnel with different credentials', () => {
  when('hashed', () => {
    const context1 = {
      aws: {
        credentials: { account: '111111111111', region: 'us-east-1' },
        ...
      },
    };
    const context2 = {
      aws: {
        credentials: { account: '222222222222', region: 'us-west-2' },
        ...
      },
    };

    hash1 = getTunnelHash({ for: { tunnel: tunnelRef } }, context1);
    hash2 = getTunnelHash({ for: { tunnel: tunnelRef } }, context2);
  });

  then('hashes should be different', () => {
    expect(hash1).not.toBe(hash2);
  });
});
```

**relation to wish**: this test already proves account differentiation works at the hash level. once `account` is in the domain object, this test should use `input.for.tunnel.account` instead of `context.aws.credentials.account`.

**action**: [EXTEND] — update tunnelRef to include account, remove context variation
