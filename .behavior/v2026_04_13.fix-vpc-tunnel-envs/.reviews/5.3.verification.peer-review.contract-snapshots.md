# blocker.1: Missing snapshot coverage for setVpcTunnel OPEN status variants

**rule**: require.contract-snapshot-exhaustiveness

**locations**:
- src/domain.operations/vpcTunnel/setVpcTunnel.test.ts

The setVpcTunnel function is a user-faced SDK method that can return tunnel objects with OPEN or CLOSED status, depending on the desired status input. The rule requires exhaustive snapshot coverage for all output variants, including positive paths, negative paths, and edge cases. However, the unit tests in setVpcTunnel.test.ts only cover CLOSED status scenarios and include snapshots for those return values. There are no unit tests or snapshots for OPEN status return values, as OPEN requires integration testing with real AWS SSM processes. This creates a gap in coverage for the OPEN positive path, which is a major output variant that callers can encounter. To fix this, add integration tests that cover OPEN status and include snapshots of the return values for those variants.
