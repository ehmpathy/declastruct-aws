# blocker.1: AWS SSM API lacks real integration tests in provided code

**rule**: require.external-contract-integration-tests

**locations**:
- src/domain.operations/vpcTunnel/getVpcTunnel.test.ts
- src/domain.operations/vpcTunnel/setVpcTunnel.test.ts

The code uses AWS SSM as an external service for VPC tunneling, but the tests in the target files mock the AWS API context instead of calling the real service. The rule requires every external contract to have at least one integration test that calls the real service with real credentials and validates responses. While comments reference integration tests elsewhere (e.g., setVpcTunnel.integration.test.ts), no such real integration tests are present in the target files. This creates false confidence as mocks can drift from reality. To fix, add integration tests that actually spawn SSM processes and verify real responses, or document a vision exception if mocking is allowed for specific reasons, but even then, at least one real test must exist.
