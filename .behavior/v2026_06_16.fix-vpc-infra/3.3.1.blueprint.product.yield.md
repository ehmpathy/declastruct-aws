# blueprint: VPC infrastructure

## summary

build 5 VPC domain objects and their operations to enable declarative VPC infrastructure provision:

1. **DeclaredAwsVpc** — virtual network container
2. **DeclaredAwsVpcSubnet** — network segment
3. **DeclaredAwsVpcSecurityGroup** — firewall rules
4. **DeclaredAwsVpcInternetGateway** — internet connectivity
5. **DeclaredAwsVpcRouteTable** — traffic routes

each domain object follows extant patterns:
- `exid` tag for unique key lookup
- `id` as metadata (AWS-assigned)
- `findsert`/`upsert` set semantics
- `genDeclastructDao` for declastruct integration

## dependency order

declastruct applies resources in dependency order. for VPC resources:

```
1. DeclaredAwsVpc                    # no dependencies
2. DeclaredAwsVpcSubnet              # depends on VPC
   DeclaredAwsVpcSecurityGroup       # depends on VPC (parallel with subnet)
   DeclaredAwsVpcInternetGateway     # depends on VPC (parallel with subnet)
3. DeclaredAwsVpcRouteTable          # depends on VPC, subnet, gateway
```

the DAO registration order in `getDeclastructAwsProvider` determines apply order.

## security group rule sync

`setVpcSecurityGroup.upsert` must sync rules:

1. get extant rules from AWS
2. compare with declared rules
3. revoke removed rules
4. authorize added rules

this is necessary because AWS security group rules are additive — an update to the security group does not replace rules, it adds to them.

## delete error allowlist

all `del*` operations use explicit error allowlist to achieve idempotency without failhide:

```typescript
try {
  await ec2.deleteVpc({ VpcId: vpcId });
} catch (error) {
  // allowlist: InvalidVpcID.NotFound → no-op (idempotent)
  if (error instanceof Error && error.name === 'InvalidVpcID.NotFound') return;
  // all other errors: rethrow (failfast)
  throw error;
}
```

this pattern:
- handles absent resources gracefully (idempotent)
- fails fast on real errors (permissions, transient failures, dependency conflicts)
- uses explicit allowlist (no catch-all)

## protocol value translation

AWS uses `-1` for "all traffic" protocol. we use readable `'all'`:

```typescript
// translate at API boundary
const awsProtocol = rule.protocol === 'all' ? '-1' : rule.protocol;
const declProtocol = awsRule.IpProtocol === '-1' ? 'all' : awsRule.IpProtocol;
```
