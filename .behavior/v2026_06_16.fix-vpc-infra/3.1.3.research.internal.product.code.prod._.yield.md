# research: internal production code patterns

## pattern.1 = domain object entity structure [REUSE]

**citation [1]**: `src/domain.objects/DeclaredAwsEc2Instance.ts`
```typescript
export interface DeclaredAwsEc2Instance {
  id?: string;          // @metadata - assigned by AWS
  exid: string;         // @unique - used for tag-based lookup
  status?: 'pending' | 'running' | ...;  // @readonly - resolved from AWS
  privateIp?: string;   // @readonly
}

export class DeclaredAwsEc2Instance extends DomainEntity<DeclaredAwsEc2Instance> {
  public static primary = ['id'] as const;
  public static unique = ['exid'] as const;
  public static metadata = ['id'] as const;
  public static readonly = ['status', 'privateIp'] as const;
}
```

**relevance**: VPC resources follow same pattern — `id` is AWS-assigned metadata, `exid` is unique key for tag-based lookup, some fields are readonly.

## pattern.7 = castInto with hasReadonly assure [REUSE]

**citation [7]**: `src/domain.operations/ec2Instance/castIntoDeclaredAwsEc2Instance.ts`
```typescript
export const castIntoDeclaredAwsEc2Instance = (
  input: Instance,
): HasReadonly<typeof DeclaredAwsEc2Instance> => {
  const exidTag = input.Tags?.find((tag) => tag.Key === 'exid');
  if (!exidTag?.Value) UnexpectedCodePathError.throw('ec2 instance lacks exid tag');

  return assure(
    DeclaredAwsEc2Instance.as({
      id: input.InstanceId,
      exid: exidTag.Value,
      status,
      privateIp: input.PrivateIpAddress,
    }),
    hasReadonly({ of: DeclaredAwsEc2Instance }),
  );
};
```

**relevance**: each VPC resource needs a `castInto*` function to convert AWS SDK response to domain object.
