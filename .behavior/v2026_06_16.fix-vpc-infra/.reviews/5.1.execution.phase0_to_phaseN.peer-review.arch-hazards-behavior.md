- src/domain.operations/vpcRouteTable/setVpcRouteTable.ts:94

Upsert operations perform multi-step mutations (revoke all current rules, then authorize desired rules; delete routes then create new routes and associations) without transactions or rollback. A failure after revoke/delete but before authorize/create leaves the resource in corrupted intermediate state (e.g., all ingress revoked but no new rules added). This produces silent data corruption that is hard to debug and only manifests on errors or retries.
