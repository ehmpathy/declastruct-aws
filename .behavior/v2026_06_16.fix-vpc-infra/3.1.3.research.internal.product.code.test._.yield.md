# research: internal test code patterns

## pattern.1 = unit test with aws-sdk-client-mock [REUSE]

**citation [1]**: `src/domain.operations/ec2Instance/getEc2Instance.test.ts`
```typescript
import { DescribeInstancesCommand, EC2Client } from '@aws-sdk/client-ec2';
import { mockClient } from 'aws-sdk-client-mock';

const ec2Mock = mockClient(EC2Client);

describe('getEc2Instance', () => {
  beforeEach(() => {
    ec2Mock.reset();
  });

  given('an instance exists with exid tag', () => {
    when('looked up by unique exid', () => {
      then('it should call DescribeInstances with tag filter', async () => {
        ec2Mock.on(DescribeInstancesCommand).resolves({
          Reservations: [{ Instances: [...] }],
        });
        result = await getEc2Instance({ by: { unique: { exid: 'test-bastion' } } }, mockContext);
      });
    });
  });
});
```

**relevance**: VPC resource unit tests use same mock pattern — `DescribeVpcsCommand`, `DescribeSubnetsCommand`, `DescribeSecurityGroupsCommand`, etc.

## pattern.8 = acceptance test with declastruct CLI [REUSE]

**citation [8]**: `src/contract/sdks/declastruct.acceptance.test.ts`
```typescript
describe('declastruct CLI workflow', () => {
  given('a declastruct resources file', () => {
    when('plan generated via declastruct CLI', () => {
      const prep = useBeforeAll(async () => {
        execSync(`npx declastruct plan --wish ${resourcesFile} --into ${planFile}`);
        return { plan: JSON.parse(readFileSync(planFile, 'utf-8')) };
      });

      then('creates a valid plan file', () => {
        expect(existsSync(planFile)).toBe(true);
      });

      then('plan structure matches snapshot', () => {
        const planSummary = prep.plan.changes.map(...);
        expect(planSummary).toMatchSnapshot();
      });
    });

    when('re-plan after apply to verify idempotency', () => {
      then('all resources show KEEP after apply', () => {
        const nonKeepChanges = prep.plan.changes.filter(r => r.action !== 'KEEP');
        expect(nonKeepChanges).toHaveLength(0);
      });
    });
  });
});
```

**relevance**: VPC resources must be added to acceptance test via `resources.acceptance.ts`.

| getMockedAwsApiContext for unit tests | [REUSE] | sync context with fake credentials |
| getSampleAwsApiContext for integration tests | [REUSE] | async context with real credentials |
| integration test with useBeforeAll | [REUSE] | same pattern for set/get operations |
| journey test for full lifecycle | [REUSE] | create → get → update → delete flow |

| resources.acceptance.ts declarations | [EXTEND] | add VPC resources to acceptance |
| snapshot test for plan structure | [REUSE] | PR vibecheck for VPC plans |

no patterns need to be replaced — all extant test patterns apply directly to VPC infrastructure test.
