## known constraints (documented, not deferred)

these items from peer review are documented constraints, not deferrals:

1. **integration tests require env var** — VPC integration tests require `DECLASTRUCT_VPC_ENABLED=true` environment variable. without it, tests fail fast with a clear error message. run with admin credentials that have VPC permissions.

2. **acceptance tests absent** — demo account lacks VPC permissions. acceptance tests are commented out with skip note. verify via dogfood provision.

3. **DeclaredAwsVpcCidrBlock v4/v6 both optional** — type represents "v4 or v6 or both". runtime validation in cast/set operations ensures at least one is present. the type system cannot express discriminated unions cleanly with DomainLiteral; the runtime behavior is correct.

4. **associationId metadata absent** — `DeclaredAwsVpcRouteTableAssociation` lacks `associationId` which AWS requires for `DisassociateRouteTable`. upsert throws `BadRequestError` when extant associations exist. to add this would require a break to the domain model. documented as feature gap.

5. **tags upsert additive-only** — tag updates only add/modify, never remove stale tags. declarative sync for tags would require diff logic. documented as feature gap for future enhancement.

6. **security group name immutable** — `name` cannot be changed after creation in AWS. upsert silently ignores name changes. documented as AWS limitation.

7. **security group rule sync non-atomic** — upsert revokes all then adds. between steps security group is empty. documented as AWS API limitation (no atomic sync available).

8. **duplicate ref dispatch in operations** — `getOneVpcId`/`getOneVpcSubnetId` manually check id/exid instead of delegating to getOne. kept for performance (avoids extra lookups). documented as intentional.
