# blackbox coverage matrix: AWS Organizations SCP support

## matrix.1 = SCP creation (findsert)

| ind: SCP name exists | ind: duplicate names | dep: result | dep: idempotent |
|----------------------|----------------------|-------------|-----------------|
| no                   | n/a                  | SCP created | yes             |
| yes (single match)   | no                   | extant returned | yes          |
| yes (multi match)    | yes                  | error raised | yes            |

notes:
- "duplicate names" means multiple SCPs with same name (created outside declastruct)
- findsert never creates duplicates — it finds or creates

## matrix.2 = SCP lookup

| ind: lookup method | ind: SCP exists | ind: duplicate names | dep: result |
|--------------------|-----------------|----------------------|-------------|
| by name            | yes             | no                   | SCP returned |
| by name            | yes             | yes                  | error raised |
| by name            | no              | n/a                  | null returned |
| by id              | yes             | n/a                  | SCP returned |
| by id              | no              | n/a                  | null returned |

notes:
- duplicate detection only applies to lookup by name
- lookup by id is unambiguous (id is primary key)

## matrix.3 = SCP attachment (findsert)

| ind: SCP exists | ind: target type | ind: attachment exists | dep: result | dep: scope |
|-----------------|------------------|------------------------|-------------|------------|
| yes             | org root         | no                     | attachment created | all accounts |
| yes             | org root         | yes                    | extant returned | all accounts |
| yes             | account          | no                     | attachment created | single account |
| yes             | account          | yes                    | extant returned | single account |
| no              | any              | n/a                    | error raised | n/a |

notes:
- target types in scope: org root (r-xxx), account (12-digit)
- OU support deferred (requires DeclaredAwsOrganizationUnit)

---

## matrix.4 = SCP detachment

| ind: attachment exists | ind: is last SCP on target | dep: result |
|------------------------|----------------------------|-------------|
| yes                    | no                         | attachment removed |
| yes                    | yes                        | error raised |
| no                     | n/a                        | no-op (idempotent) |

notes:
- AWS requires at least 1 SCP per entity when SCPs are enabled
- cannot detach the last SCP from a target

## gap analysis

no gaps detected. all meaningful combinations are covered:

1. **SCP creation**: all existence states covered
2. **SCP lookup**: both lookup methods covered, with duplicate detection
3. **SCP attachment**: both target types covered, idempotent
4. **SCP detachment**: last-SCP protection covered
5. **SCP deletion**: attachment constraint covered
6. **Validation**: boundary conditions covered

---

## decomposition analysis

current scope is well-bounded:
- 2 independent variables per matrix (manageable)
- clear separation between SCP and attachment concerns
- no explosion of combinations

OU support (deferred) would add one more target type but not increase complexity significantly.
