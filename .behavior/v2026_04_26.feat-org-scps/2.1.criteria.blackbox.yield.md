# blackbox criteria: AWS Organizations Service Control Policy support

## usecase.1 = create and manage service control policies

### episode: create a new SCP

given a security engineer with org management account credentials
  when they declare an SCP with name, content, and optional description/tags
    then the SCP is created in their organization
      sothat they can define org-wide permission boundaries
    then the SCP is retrievable by its name
      sothat they can verify it was created correctly
    then the SCP has an assigned id and arn
      sothat they can reference it in other systems

given an SCP with that name already exists
  when they declare an SCP with the same name
    then the extant SCP is returned
      sothat the operation is idempotent

### episode: look up SCPs

given SCPs exist in the organization
  when they look up an SCP by name
    then the SCP is returned with all its attributes
      sothat they can inspect its current state
  when they look up an SCP by id
    then the SCP is returned
      sothat they can retrieve by primary key

given no SCP exists with that name
  when they look up by name
    then null is returned
      sothat they know it does not exist

given duplicate SCPs with the same name exist (created outside declastruct)
  when they look up by name
    then an error is raised
      sothat silent wrong-selection is prevented

## usecase.2 = attach SCPs to targets

### episode: attach SCP to org root

given an SCP exists
  when they declare an attachment to the org root
    then the SCP applies to all accounts in the organization
      sothat org-wide guardrails are enforced

given the attachment already exists
  when they declare the same attachment again
    then the extant attachment is returned
      sothat the operation is idempotent

### episode: attach SCP to specific account

given an SCP exists
  when they declare an attachment to a specific account
    then the SCP applies only to that account
      sothat targeted guardrails can be enforced

---

## usecase.3 = detach and delete SCPs

### episode: detach SCP from target

given an SCP is attached to a target
  when they remove the attachment
    then the SCP no longer applies to that target
      sothat guardrails can be narrowed

given the SCP is the last one attached to the target
  when they attempt to detach it
    then an error is raised
      sothat the minimum SCP requirement is preserved

### episode: delete SCP

given an SCP exists and is not attached anywhere
  when they delete the SCP
    then the SCP is removed from the organization
      sothat unused policies can be cleaned up

given an SCP is still attached to targets
  when they attempt to delete it
    then an error is raised
      sothat active policies are not accidentally removed

---

## usecase.4 = enforce org-wide security guardrails

### episode: block data exfiltration

given an SCP denies snapshot share actions
  when the SCP is attached to org root
    then no principal in any account can share RDS/EBS snapshots externally
      sothat data exfiltration via snapshot is blocked
    then no principal can share AMIs externally
      sothat data exfiltration via AMI is blocked

### episode: protect audit trail

given an SCP denies audit tamper actions
  when the SCP is attached to org root
    then no principal can delete or stop CloudTrail
      sothat audit logs are protected
    then no principal can disable Config or GuardDuty
      sothat security detection is preserved

### episode: prevent org escape

given an SCP denies LeaveOrganization
  when the SCP is attached to org root
    then no account can leave the organization
      sothat guardrails cannot be bypassed

---

## exchange.1 = SCP creation contract

input:
  - name: string (1-128 chars, required)
  - content: policy document (max 5,120 chars serialized, required)
  - description: string (max 512 chars, optional)
  - tags: key-value pairs (optional)

output:
  - id: string (e.g., 'p-abc123')
  - arn: string (e.g., 'arn:aws:organizations::123:policy/...')
  - name: string
  - content: policy document
  - description: string or undefined
  - tags: key-value pairs or undefined

errors:
  - name exceeds 128 chars
  - content exceeds 5,120 chars
  - invalid policy document syntax
  - duplicate names detected in organization

---

## exchange.2 = SCP attachment contract

input:
  - policy: reference by name
  - target: reference by id (org root 'r-xxx' or account '123456789012')

output:
  - policy: reference
  - target: reference

errors:
  - policy not found
  - target not found
  - max attachments exceeded (5 per target)
  - would result in no Allow statements (effective deny-all)

---

## exchange.3 = SCP lookup contract

input:
  - by unique: { name: string }
  - or by primary: { id: string }

output:
  - SCP with all attributes, or null if not found

errors:
  - duplicate names detected (for lookup by name)

---

## exchange.4 = SCP deletion contract

input:
  - policy: reference by name or id

output:
  - void (success)

errors:
  - policy not found
  - policy still attached to targets
