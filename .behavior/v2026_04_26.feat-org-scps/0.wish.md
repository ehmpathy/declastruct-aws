# handoff: declastruct-aws service control policy support

## context

ahbode uses aws organizations with multiple accounts (root, prep, prod). scps provide org-level guardrails that apply to all principals in an account — even those with AdministratorAccess.

without scps, a compromised role with PowerUserAccess can:
- share rds/ebs snapshots to external accounts (exfiltration)
- disable cloudtrail (cover tracks)
- share amis externally

scps block these actions at the org level, regardless of iam permissions.

## what we need

### DeclaredAwsOrganizationServiceControlPolicy

| field | type | description |
|-------|------|-------------|
| `name` | `string` | policy name |
| `description` | `string \| null` | policy description |
| `content` | `DeclaredAwsIamPolicyDocument` | the policy document |
| `tags` | `Record<string, string> \| null` | resource tags |

### DeclaredAwsOrganizationServiceControlPolicyAttachment

| field | type | description |
|-------|------|-------------|
| `policy` | `RefByUnique<DeclaredAwsOrganizationServiceControlPolicy>` | which policy |
| `target` | `RefByPrimary<DeclaredAwsOrganization \| DeclaredAwsOrganizationAccount>` | org root or account |

## example usage

```typescript
import {
  DeclaredAwsOrganizationServiceControlPolicy,
  DeclaredAwsOrganizationServiceControlPolicyAttachment,
  DeclaredAwsIamPolicyDocument,
  DeclaredAwsIamPolicyStatement,
  DeclaredAwsOrganization,
} from 'declastruct-aws';

// deny dangerous actions org-wide
const denyDangerousActions = DeclaredAwsOrganizationServiceControlPolicy.as({
  name: 'deny-dangerous-actions',
  description: 'block exfiltration, audit tamper, and escape vectors',
  content: new DeclaredAwsIamPolicyDocument({
    statements: [
      // block exfiltration via snapshot share
      new DeclaredAwsIamPolicyStatement({
        sid: 'DenySnapshotShare',
        effect: 'Deny',
        action: [
          'rds:ModifyDBSnapshotAttribute',
          'rds:ModifyDBClusterSnapshotAttribute',
          'ec2:ModifySnapshotAttribute',
          'ec2:ModifyImageAttribute',
        ],
        resource: '*',
      }),
      // block audit tamper
      new DeclaredAwsIamPolicyStatement({
        sid: 'DenyAuditTamper',
        effect: 'Deny',
        action: [
          'cloudtrail:DeleteTrail',
          'cloudtrail:StopLogging',
          'cloudtrail:UpdateTrail',
          'config:DeleteConfigRule',
          'config:StopConfigurationRecorder',
          'guardduty:DeleteDetector',
        ],
        resource: '*',
      }),
      // block org escape
      new DeclaredAwsIamPolicyStatement({
        sid: 'DenyOrgEscape',
        effect: 'Deny',
        action: ['organizations:LeaveOrganization'],
        resource: '*',
      }),
    ],
  }),
  tags: {
    managedBy: 'declastruct',
  },
});

// attach to org root (applies to all accounts)
const attachment = DeclaredAwsOrganizationServiceControlPolicyAttachment.as({
  policy: refByUnique(denyDangerousActions),
  target: refByUnique(organization),
});
```

## aws api reference

**create policy:**
```
POST /organizations/CreatePolicy
{
  "Content": "{...}",
  "Description": "...",
  "Name": "...",
  "Type": "SERVICE_CONTROL_POLICY"
}
```

**attach policy:**
```
POST /organizations/AttachPolicy
{
  "PolicyId": "p-xxx",
  "TargetId": "r-xxx" | "ou-xxx" | "123456789012"
}
```

**docs:** https://docs.aws.amazon.com/organizations/latest/APIReference/API_CreatePolicy.html

## scps ahbode needs

| scp | blocks | why |
|-----|--------|-----|
| deny-snapshot-share | rds/ebs/ami share to external accounts | prevents data exfiltration |
| deny-audit-tamper | cloudtrail/config/guardduty modification | prevents track cover |
| deny-org-escape | leave organization | prevents guardrail escape |
| enforce-region (optional) | actions outside us-east-1, us-west-2 | reduces attack surface |

## why scps over iam denies

| aspect | iam deny (per role) | scp (org level) |
|--------|---------------------|-----------------|
| scope | one role | all principals in account |
| bypass | create new role | impossible |
| maintenance | per role | one policy |
| applies to | assumed role | all principals and root user |

scps set the upper bound — iam permissions can only be equal or more restrictive.
