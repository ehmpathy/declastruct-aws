# blocker.1: Missing exhaustive snapshot coverage for declastruct CLI and SDK contracts

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.verification/rule.require.contract-snapshot-exhaustiveness.md

**locations**:
- src/contract/sdks/declastruct.acceptance.test.ts

The acceptance test for declastruct CLI workflow and SDK methods lacks exhaustive snapshot coverage for all output variants as required by the rule. The rule mandates snapshots for positive paths, negative paths, help outputs (for CLI), and edge cases to ensure zero gaps in caller experience. Currently, the test uses toMatchSnapshot() for plan structures in successful scenarios but does not capture or snapshot CLI stdout/stderr for error cases, invalid inputs, or help commands. Similarly, SDK method calls (e.g., DAO operations for log group reports and IAM access keys) assert on results without snapshotting return values for all variants like found, not-found, invalid inputs, or edge cases. This creates blind spots in review and regression detection. To comply, add explicit snapshots for all CLI command outputs (stdout/stderr) across variants, and snapshot SDK method return values for each possible outcome.
