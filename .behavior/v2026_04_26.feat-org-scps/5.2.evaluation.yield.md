## divergence analysis

### divergences found

| section | blueprint declared | actual implemented | divergence type | severity |
|---------|-------------------|-------------------|-----------------|----------|
| organization | rootId as readonly field | rootId removed, getOrganizationRootId added | changed | enhancement |
| SCP operations | separate integration tests per operation | single journey test covers all | consolidated | acceptable |
| attachment target | target: RefByPrimary<Org \| Account> | target: { id } \| { rootId } pattern | changed | enhancement |
| helper transformers | not declared | asTargetId, isOrgRootRef added | added | enhancement |

### divergence decision

| divergence | decision | rationale |
|------------|----------|-----------|
| rootId refactor | **accepted** | rootId is not intrinsic to Organization (not returned by DescribeOrganization). creating a lazy-fetch pattern via getOrganizationRootId is cleaner and avoids scope leak. this is an architectural improvement over the blueprint. |
| consolidated integration tests | **accepted** | a journey test covering the full lifecycle is more comprehensive than isolated tests. all codepaths are exercised in sequence, testing real-world usage. matches extant patterns (e.g., iamRole tests). |
| target pattern | **accepted** | using `{ id }` for account and `{ rootId }` for org root makes the API more explicit about what type of target is being referenced. prevents confusion between org id (o-xxx) and root id (r-xxx). |
| helper transformers | **accepted** | `asTargetId` and `isOrgRootRef` are necessary decompositions to handle the target polymorphism cleanly. follows transformer pattern from architect briefs. |

---

## conclusion

all divergences are enhancements that improve the implementation over the blueprint. no blockers.

- rootId refactor: cleaner architecture, avoids scope leak
- consolidated tests: comprehensive journey coverage
- target pattern: explicit API, prevents id confusion
- helper transformers: follows decomposition patterns

implementation adheres to the blueprint intent while making justified improvements.
