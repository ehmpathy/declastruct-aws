# research: internal prod code patterns for AWS Organizations SCP support

## summary

this document catalogs the key patterns in declastruct-aws that are relevant to SCP support. each pattern is marked [REUSE], [EXTEND], or [REPLACE].

---

## pattern.1: domain object declaration

### [REUSE] DomainEntity for resources with AWS-assigned identity

entities with `id` or `arn` assigned by AWS extend `DomainEntity`.

> "export class DeclaredAwsOrganization extends DomainEntity<DeclaredAwsOrganization> implements DeclaredAwsOrganization" [1]

static fields:
- `.primary` = primary key (aws-assigned id)
- `.unique` = natural key (user-specified)
- `.metadata` = fields assigned by aws
- `.readonly` = fields not settable by user
- `.nested` = nested domain object types

citation [1]: src/domain.objects/DeclaredAwsOrganization.ts:69-100

### [REUSE] DomainLiteral for value objects without identity

value objects like policy documents extend `DomainLiteral`.

> "export class DeclaredAwsIamPolicyDocument extends DomainLiteral<DeclaredAwsIamPolicyDocument> implements DeclaredAwsIamPolicyDocument" [2]

citation [2]: src/domain.objects/DeclaredAwsIamPolicyDocument.ts:20-27

### [REUSE] reference types for foreign keys

`RefByPrimary<T>` for primary key references, `RefByUnique<T>` for natural key references.

> "managementAccount: RefByPrimary<typeof DeclaredAwsOrganizationAccount>" [3]

> "organization: RefByUnique<typeof DeclaredAwsOrganization>" [4]

citation [3]: src/domain.objects/DeclaredAwsOrganization.ts:59
citation [4]: src/domain.objects/DeclaredAwsOrganizationAccount.ts:58

## pattern.5: declastruct DAO

### [REUSE] genDeclastructDao factory

the DAO factory from declastruct package.

> "export const DeclaredAwsOrganizationDao = genDeclastructDao<typeof DeclaredAwsOrganization, ContextAwsApi & ContextLogTrail>({ dobj: DeclaredAwsOrganization, get: { one: { byPrimary: ..., byUnique: ... } }, set: { findsert: ..., upsert: ..., delete: ... } })" [12]

structure:
- `dobj` = domain object class
- `get.one.byPrimary` = lookup by primary key
- `get.one.byUnique` = lookup by unique key
- `set.findsert` = idempotent create
- `set.upsert` = create or update
- `set.delete` = remove

citation [12]: src/access/daos/DeclaredAwsOrganizationDao.ts:17-41

### [REUSE] attachment DAO pattern

for attachments without primary key, `byPrimary` is null.

> "get: { one: { byPrimary: null, byUnique: async (input, context) => { return getIamRolePolicyAttachedManaged({ by: { unique: input } }, context); } } }" [13]

citation [13]: src/access/daos/DeclaredAwsIamRolePolicyAttachedManagedDao.ts:20-29

## pattern.6: domain operations

### [REUSE] getOne* operation pattern

lookup operations support multiple lookup methods via `PickOne`.

> "input: { by: PickOne<{ auth: true; primary: RefByPrimary<typeof DeclaredAwsOrganization>; unique: RefByUnique<typeof DeclaredAwsOrganization>; ref: Ref<typeof DeclaredAwsOrganization>; }>; }" [14]

type guards for ref dispatch:
> "if (isRefByUnique({ of: DeclaredAwsOrganization })(input.by.ref)) return getOneOrganization({ by: { unique: input.by.ref } }, context);" [14]

citation [14]: src/domain.operations/organization/getOneOrganization.ts:31-48

### [REUSE] set* operation pattern

set operations support findsert and/or upsert via `PickOne`.

> "input: PickOne<{ findsert: DeclaredAwsOrganization; }>" [15]

findsert pattern: lookup first, return if found, create if not.
> "const foundBefore = await getOneOrganization({ by: { unique: ... } }, context); if (foundBefore) return foundBefore;" [15]

citation [15]: src/domain.operations/organization/setOrganization.ts:25-62

### [REUSE] del* operation pattern

delete operations return `{ deleted: true }` and are idempotent.

> "if (!foundBefore) return { deleted: true };" [16]

citation [16]: src/domain.operations/organization/delOrganization.ts:57-58

### [REUSE] castInto* transformer pattern

transformers convert AWS SDK types to domain objects.

> "export const castIntoDeclaredAwsOrganization = (input: Organization): HasReadonly<typeof DeclaredAwsOrganization> => { return assure(DeclaredAwsOrganization.as({ ... }), hasReadonly({ of: DeclaredAwsOrganization })); }" [17]

citation [17]: src/domain.operations/organization/castIntoDeclaredAwsOrganization.ts:15-29

## application to SCP support

### DeclaredAwsOrganizationServiceControlPolicy

[REUSE] patterns 1, 3 for domain object structure:
- extend `DomainEntity`
- primary = [id] (assigned by aws)
- unique = [name] (enforced by aws)
- metadata = [id, arn]
- content field reuses `DeclaredAwsIamPolicyDocument`

### DeclaredAwsOrganizationServiceControlPolicyAttachment

[EXTEND] pattern 4 for attachment structure:
- extend `DomainEntity` (like managed policy attachment)
- unique = [policy, target] (composite key)
- no primary key (attachments have no aws-assigned id)
- policy via `RefByUnique` (lookup by name)
- target via `Ref` (supports org root or account)

### daoAwsOrganizationServiceControlPolicy

[REUSE] pattern 5 for DAO structure:
- `genDeclastructDao` factory
- get.one.byPrimary = lookup by id
- get.one.byUnique = lookup by name
- set.findsert = idempotent create
- set.delete = detach all then delete

### daoAwsOrganizationServiceControlPolicyAttachment

[EXTEND] pattern 5 for attachment DAO:
- `genDeclastructDao` factory
- get.one.byPrimary = null (no primary key)
- get.one.byUnique = lookup by [policy, target]
- set.findsert = idempotent attach
- set.delete = detach (with last-SCP protection)

### domain operations

[REUSE] patterns 6, 7, 9 for operations:
- getOneOrganizationServiceControlPolicy
- setOrganizationServiceControlPolicy
- delOrganizationServiceControlPolicy
- castIntoDeclaredAwsOrganizationServiceControlPolicy
- extend getAwsOrganizationsClient (already exists)
