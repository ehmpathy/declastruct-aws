# research: internal test code patterns for AWS Organizations SCP support

## summary

this document catalogs the key test patterns in declastruct-aws that are relevant to SCP support. each pattern is marked [REUSE], [EXTEND], or [REPLACE].

---

## pattern.1: unit test structure

### [REUSE] BDD-style tests with test-fns

unit tests use `given`, `when`, `then` from test-fns package.

> "import { given, then, when } from 'test-fns';" [1]

structure:
```typescript
describe('operationName', () => {
  given('scenario description', () => {
    when('action description', () => {
      then('expected outcome', async () => { ... });
    });
  });
});
```

citation [1]: src/domain.operations/organization/getOneOrganization.test.ts:5

### [REUSE] jest.mock for SDK dependencies

AWS SDK clients and internal dependencies are mocked at module level.

> "jest.mock('@aws-sdk/client-organizations');" [2]
> "jest.mock('./castIntoDeclaredAwsOrganization');" [2]

citation [2]: src/domain.operations/organization/getOneOrganization.test.ts:12-13

### [REUSE] mock send pattern

SDK client's `send` method is mocked to control responses.

> "const mockSend = jest.fn();" [3]
> "(OrganizationsClient as jest.Mock).mockImplementation(() => ({ send: mockSend }));" [3]

citation [3]: src/domain.operations/organization/getOneOrganization.test.ts:15-18

### [REUSE] beforeEach clearAllMocks

mocks are cleared between tests for isolation.

> "beforeEach(() => { jest.clearAllMocks(); });" [4]

citation [4]: src/domain.operations/organization/getOneOrganization.test.ts:23-25

## application to SCP support

### DeclaredAwsOrganizationServiceControlPolicy tests

[REUSE] patterns 1, 2, 6 for domain object tests:
- instantiation tests
- static key tests (primary, unique, metadata)

[REUSE] pattern 4 for cast function test:
- happy path cast
- absent field tests

### getOneOrganizationServiceControlPolicy tests

[REUSE] patterns 1, 2, 3 for operation tests:
- mock OrganizationsClient send
- test by primary lookup
- test by unique lookup
- test null when not found
- test error wrap

### setOrganizationServiceControlPolicy tests

[REUSE] patterns 1, 2, 3 for operation tests:
- test findsert creates when not found
- test findsert returns extant when found (idempotent)
- test validation errors

### delOrganizationServiceControlPolicy tests

[REUSE] patterns 1, 2, 3 for operation tests:
- test delete succeeds
- test idempotent when not found
- test constraint error when attached

### DeclaredAwsOrganizationServiceControlPolicyAttachment tests

[EXTEND] patterns 1, 6 for domain object tests:
- instantiation tests
- composite unique key tests

[EXTEND] pattern 7 for attachment operation tests:
- test findsert creates attachment
- test findsert idempotent when extant
- test detach constraint (last SCP protection)

### integration tests

[EXTEND] pattern 5 for integration tests:
- test against real AWS Organizations API
- requires org management account credentials
- test full findsert + attach + detach + delete cycle
